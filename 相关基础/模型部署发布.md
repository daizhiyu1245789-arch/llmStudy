# 模型部署与发布：从训练到让别人用上你的模型

> 训练好的模型只是一个 `.pt` 文件，怎么变成别人能用的东西？
> 这一篇覆盖：直接用、API 服务化、打包成分发格式、模型市场发布。

## 1. 最简单的方式：直接分享 .pt 文件

训练完成后，`checkpoints/best.pt` 就是完整的模型文件。

```bash
# 压缩一下，方便传输
zip best_model.zip checkpoints/best.pt

# 别人拿到后，直接用 generate.py 加载
python -m tinyllm.generate --ckpt_path best_model.pt --prompt "今天" --max_new_tokens 100
```

**优点**：简单直接，零门槛
**缺点**：对方必须会 Python、会跑脚本，不适合普通用户

## 2. 封装成 API 服务：让模型变成"接口"

这是最常见的生产级部署方式。

### 2.1 用 Flask / FastAPI 包装成本地 API

```python
# server.py
from flask import Flask, request, jsonify
import torch
from tinyllm.model import GPT, GPTConfig
from tinyllm.tokenizer import tokenizer_from_extra, tokenizer_to_extra
from tinyllm.checkpoint import load_checkpoint

app = Flask(__name__)

# 启动时加载模型（只加载一次，不重复加载）
ckpt = load_checkpoint("checkpoints/best.pt", map_location="cpu")
tokenizer = tokenizer_from_extra(ckpt.get("extra"))
model = GPT(GPTConfig(**ckpt["model_cfg"]))
model.load_state_dict(ckpt["model_state"], strict=True)
model.eval()

@app.route("/generate", methods=["POST"])
def generate():
    prompt = request.json.get("prompt", "")
    max_tokens = request.json.get("max_tokens", 100)
    temperature = request.json.get("temperature", 1.0)
    top_k = request.json.get("top_k", 50)

    ids = tokenizer.encode(prompt)
    if not ids:
        ids = [0]
    idx = torch.tensor([ids], dtype=torch.long)

    out = model.generate(
        idx,
        max_new_tokens=max_tokens,
        temperature=temperature,
        top_k=top_k if top_k > 0 else None,
    )

    text = tokenizer.decode(out[0].tolist())
    return jsonify({"text": text})

if __name__ == "__main__":
    app.run(host="0.0.0.0", port=3000)
```

```bash
# 启动服务
python server.py
```

```js
// 前端调用（Node.js / 浏览器 fetch）
const response = await fetch("http://localhost:3000/generate", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({
    prompt: "今天天气",
    max_tokens: 100,
    temperature: 0.9,
    top_k: 50
  })
});
const { text } = await response.json();
console.log(text); // "今天天气很不错，适合出门散步..."
```

**类比**：
```
这个 API 服务就像你写的一个 Express 接口
  POST /generate
  → 接收 JSON body（prompt）
  → 返回 JSON { text: "生成结果" }
  → 前端用 fetch/axios 调用
```

## 3. 部署到云服务器 / 云函数

### 3.1 云服务器（ECS / 云主机）

```bash
# 1. 把项目传到服务器
scp -r tinyllm checkpoints/ user@your-server:/home/user/

# 2. SSH 进去，安装依赖
ssh user@your-server
pip install torch numpy tqdm
cd tinyllm

# 3. 启动服务（后台运行）
nohup python server.py > server.log 2>&1 &
# nohup：关掉终端也在跑
# &：后台运行

# 4. 验证服务在跑
curl -X POST http://localhost:3000/generate \
  -H "Content-Type: application/json" \
  -d '{"prompt": "今天"}'
```

### 3.2 无服务器云函数（Serverless）

适合调用量不大的场景，按调用次数计费，不用自己维护服务器。

```python
# 腾讯云 SCF / 阿里云 FC / AWS Lambda 写法类似
def main_handler(request, context):
    import torch
    from tinyllm.model import GPT, GPTConfig
    from tinyllm.checkpoint import load_checkpoint
    from tinyllm.tokenizer import tokenizer_from_extra

    # 冷启动时加载模型（云函数会复用实例）
    ckpt = load_checkpoint("/tmp/best.pt", map_location="cpu")
    tokenizer = tokenizer_from_extra(ckpt.get("extra"))
    model = GPT(GPTConfig(**ckpt["model_cfg"]))
    model.load_state_dict(ckpt["model_state"])
    model.eval()

    body = request.json
    prompt = body.get("prompt", "")
    # ... 生成逻辑同上
    return {"text": text}
```

**缺点**：云函数冷启动慢（第一次加载模型要 10-30 秒），适合调用量不高的场景。

## 4. 用 ONNX 格式跨平台分发

`.pt` 文件只能在 Python 里用。**ONNX**（Open Neural Network Exchange）可以把模型转为跨平台的通用格式。

```python
# 导出为 ONNX 格式
import torch

ckpt = load_checkpoint("checkpoints/best.pt", map_location="cpu")
model = GPT(GPTConfig(**ckpt["model_cfg"]))
model.load_state_dict(ckpt["model_state"])

# 构造一个 dummy input
dummy = torch.randint(0, 256, (1, 10))

# 导出
torch.onnx.export(
    model,
    dummy,
    "gpt_model.onnx",
    input_names=["input_ids"],
    output_names=["logits"],
    dynamic_axes={"input_ids": {0: "batch", 1: "seq_len"}}
)
```

```
.pt 文件（只能用 Python + PyTorch）
    ↓ torch.onnx.export
.onnx 文件（可以用多种语言加载）
    ↓
C++ / C# / Java / JavaScript（TensorFlow.js）/ Go 都能用
```

## 5. 前端直接用：TensorFlow.js / Transformers.js

如果想让用户在浏览器里直接跑模型，不需要服务器。

### 5.1 Transformers.js（推荐，Hugging Face 出品）

```html
<!-- 浏览器里直接用，不需要后端 -->
<script type="module">
  import { pipeline } from "https://cdn.jsdelivr.net/npm/@xenova/transformers";

  const generator = await pipeline("text-generation", "gpt2");

  const result = await generator("今天天气", {
    max_new_tokens: 50,
    temperature: 0.9,
  });

  console.log(result[0].generated_text);
</script>
```

### 5.2 自己训练的模型怎么转成浏览器能用的格式？

本项目的 GPT 是 PyTorch 模型，Hugging Face 的 `transformers` 库支持导入：

```python
# 先把本项目的 GPT 包装成 Hugging Face 格式
# 1. 安装 transformers
pip install transformers

# 2. 创建一个 wrapper 把 tinyllm 的 GPT 适配成 HF 的 AutoModelForCausalLM 接口
# 这需要实现一些标准方法（from_pretrained, save_pretrained, forward 等）

# 3. 保存为 HF 格式
model.save_pretrained("./hf_model")
tokenizer.save_pretrained("./hf_model")

# 4. 上传到 Hugging Face Hub，或者转成 ONNX 再转成 TensorFlow.js
```

### 5.3 完整流程：PyTorch → ONNX → TensorFlow.js → 前端

```bash
# Step 1: PyTorch → ONNX
python export_onnx.py --ckpt checkpoints/best.pt --output gpt.onnx

# Step 2: ONNX → TensorFlow.js
pip install onnx-tf
onnx-tf convert -t tf -i gpt.onnx -o tf_model/
tensorflowjs_converter --input_format=tf_saved_model --output_format=tensorflowjs_graph_model tf_model/ web_model/

# Step 3: 前端加载
```

```html
<script src="https://cdn.jsdelivr.net/npm/@tensorflow/tfjs"></script>
<script src="https://cdn.jsdelivr.net/npm/@tensorflow/tfjs-converter"></script>
<script>
  async function loadModel() {
    const model = await tf.loadGraphModel("web_model/model.json");
    // ... 调用模型进行推理
  }
</script>
```

**限制**：本项目的模型虽然小（几 MB），但浏览器里跑推理依然很慢（没有 GPU 加速）。真正要跑，还是得用后端 API。

### 6.1 发布到 Hugging Face Hub

```bash
pip install huggingface_hub
```

## 7. 总结：选择哪种方式？

| 场景 | 推荐方式 |
|------|---------|
| 自己用 / 小范围分享 | 直接给 .pt 文件 + generate.py 脚本 |
| 给开发者用，提供 API | FastAPI 包装成 REST API，部署到云服务器 |
| 大量用户访问 | Docker 容器化，部署到 K8s 集群，配 GPU 实例 |
| 前端页面直接集成 | ONNX → TensorFlow.js（但有性能限制） |
| 开源项目 / 学术分享 | Hugging Face Hub / GitHub Release |
| 免费且调用量小 | 无服务器云函数（按调用计费） |

**最实用的生产路径**：

```
训练产出 best.pt
    ↓
FastAPI 包装成 /generate 接口
    ↓
Docker 打包
    ↓
部署到云服务器（带 GPU 的更好，没有也行，CPU 慢一点）
    ↓
前端通过 HTTP 调用
```
