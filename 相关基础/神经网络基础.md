# 神经网络基础概念

> 深度学习的一切都建立在"神经元"和"层"的概念上。本篇用搭积木的方式解释这些概念。

---

## 1. 什么是神经元？—— 就像 Express 的一个路由处理函数

一个最简单的神经元做的事情就是：

```
输入 × 权重 + 偏置 → 激活函数 → 输出
```

类比 Node.js：

```js
// 单个"神经元"
function neuron(input, weight, bias) {
  const raw = input * weight + bias; // 线性变换
  const activated = Math.tanh(raw);   // 激活函数（类似中间件的 filter）
  return activated;
}
```

**激活函数是什么？**
- 没有激活函数，`raw = input * weight + bias` 就是纯线性运算
- 再多层线性变换叠加，结果还是线性的（相当于一层）
- **激活函数引入非线性**，让神经网络能表达复杂模式

常见激活函数：
- **ReLU**：`max(0, x)` — 最常用，就像过滤掉负数
- **GELU**：本项目用的，平滑版的 ReLU
- **Softmax**：把一堆数字变成概率分布（和为 1）

## 2. 什么是层（Layer）？—— 就像 Express 的一个中间件

一层 = **很多个神经元并排**，每个神经元独立处理输入：

```
输入 [x1, x2, x3]
      ↓
  神经元1 → y1
  神经元2 → y2   （这一层有 3 个神经元，输出 3 个数）
  神经元3 → y3
      ↓
输出 [y1, y2, y3]
```

```js
// 密集层（Dense Layer）= 全连接层 = 一排神经元并行计算
function denseLayer(input, weights, biases, activation) {
  const outputs = [];
  for (let i = 0; i < weights.length; i++) {
    let sum = biases[i];
    for (let j = 0; j < input.length; j++) {
      sum += input[j] * weights[i][j];
    }
    outputs.push(activation(sum));
  }
  return outputs;
}
```

**本项目的层结构**：

```
Embedding 层      → (vocab_size=256) × (n_embd=128) = 32768 个参数
Position Embedding → (block_size=128) × (n_embd=128) = 16384 个参数
Transformer Block × 4 → 每块包含：Attention(×2) + MLP(×2) + LayerNorm(×2)
  - Attention: QKV投影 3×(128×384) + 输出投影 384×128
  - MLP: 128×512 + 512×128
  - LayerNorm: 2×128×2
最终输出层 (LM Head) → 128 × 256（但权重与 Embedding 共享）
```

## 3. 什么是"前向传播"？—— 就像 Koa 的中间件调用链

```js
// Koa 的中间件栈
app.use(middleware1);
app.use(middleware2);
app.use(middleware3);

// 请求来的时候：
//   ctx → middleware1 → middleware2 → middleware3 → ctx.body = 响应
```

前向传播（Forward Propagation）就是数据从输入流到输出的过程：

```
输入 Token ID
  ↓
Embedding 层（查表得到向量）
  ↓
位置编码相加
  ↓
Dropout（随机丢弃一些神经元）
  ↓
Block 1: LayerNorm → Attention → Add → LayerNorm → MLP → Add
  ↓
Block 2: (同上)
  ↓
Block 3: (同上)
  ↓
Block 4: (同上)
  ↓
LayerNorm → LM Head（输出词表概率）
  ↓
计算损失（与目标对比）
```

每一步都是**确定的**（给定输入，输出固定），不涉及梯度计算。

## 4. 什么是"反向传播"？—— 就像 Bug 的错误栈追踪

```js
// Node.js 出错时的错误栈
Error: Cannot read property 'name' of undefined
    at UserService.getName (/app/service.js:42:18)   ← 根源
    at UserController.handler (/app/controller.js:15:10)
    at Router.dispatch (/app/router.js:8:12)
```

反向传播（Backpropagation）就是一个**反向的错误追踪过程**：

```
最终输出错了（损失 = 0.8，很高）
  ↑
LayerNorm 层贡献了 0.1 的错误
  ↑
Block 4 的 Attention 贡献了 0.2 的错误
  ↑
Block 4 的 MLP 贡献了 0.15 的错误
  ↑
Block 3 的 Attention 贡献了 0.2 的错误
  ↑
Block 1 的 Embedding 贡献了 0.15 的错误
  ↓
从最终的损失，一路追溯到每个参数应该怎么调整
```

**链式法则（Chain Rule）**是反向传播的数学基础：
- 复合函数的梯度 = 各部分梯度的乘积
- `dL/dW = dL/dy * dy/dW`

## 5. 什么是"梯度"？—— 就像斜坡的倾斜程度

```
      ↗ 梯度大（陡峭）
    ↗
山坡 ↗          ↘
    ↘            ↘ 梯度小（平缓）
      ↘          ↘
```

梯度（Gradient）就是一个向量，**指向函数值增长最快的方向**。

- **下山找最小值**：沿着梯度的**反方向**走
- **梯度越大**：参数稍微动一点，损失就变化很大
- **梯度趋近 0**：我们到达了"谷底"（损失最低点）

```js
// 梯度下降伪代码
for (let i = 0; i < steps; i++) {
  gradient = computeGradient(loss, params); // 算梯度
  params = params - learningRate * gradient; // 往梯度的反方向走一步
}
```

## 6. 什么是"学习率"？—— 就像跑步时每步迈多远

| 学习率 | 效果 | 类比 |
|--------|------|------|
| 太大（0.1） | 在最低点附近来回跳，永远到不了 | 一步迈 10 米，在山谷两侧跳来跳去 |
| 太小（1e-6） | 收敛极慢，可能卡在局部最优 | 一步走 1 毫米，下辈子也到不了 |
| 刚刚好（3e-4） | 稳定收敛到最优解 | 一步走 0.3 米，有节奏地下山 |

本项目默认 `learning_rate = 3e-4`（0.0003），是一个经过大量实验验证的"黄金值"。

## 7. 什么是"Dropout"？—— 就像压测时随机关掉一些服务器

生产环境做压测时，你随机关掉一些服务节点：

```js
// 模拟 Dropout：随机关闭 10% 的"神经元"
const activeServers = allServers.filter(() => Math.random() > 0.1);
const results = activeServers.map(s => s.handle(request));
```

Dropout 的作用：
- **训练时**：随机关掉 10% 的神经元，强迫网络不依赖特定神经元（正则化）
- **推理时**：所有神经元都参与（Dropout 自动关闭）

## 8. 什么是"LayerNorm"？—— 就像把数据做标准化

```js
// LayerNorm 的简化理解：
// 对一层的输出，做标准化：减去均值，除以标准差
function layerNorm(x) {
  const mean = x.reduce((a, b) => a + b) / x.length;
  const std = Math.sqrt(x.map(v => (v - mean) ** 2).reduce((a, b) => a + b) / x.length);
  return x.map(v => (v - mean) / std); // 标准化
}
```

作用：
- 让每层的输入分布稳定（不那么"跑偏"）
- 加速训练收敛
- 类似于批标准化（BatchNorm），但 LayerNorm 不依赖 batch size

## 9. 什么是"过拟合"和"欠拟合"？

```
训练损失:    1.2  →  0.3  →  0.1  →  0.05  (持续下降)
验证损失:    1.5  →  1.4  →  1.6  →  2.5   (先降后升 ← 过拟合！)
```

| | 欠拟合（Underfitting） | 过拟合（Overfitting） |
|---|---|---|
| 表现 | 训练和验证损失都很高 | 训练损失低，验证损失高 |
| 原因 | 模型太简单 / 训练不够 | 模型太复杂 / 训练过头 |
| 解决 | 增大模型 / 多训练 | 减少层数 / 加 Dropout / 加数据 |

## 10. 什么是"AdamW"？—— 梯度下降的"升级版"

普通的梯度下降（SGD）像这样更新参数：

```js
// SGD：直接减去梯度
weight = weight - learningRate * gradient;
```

AdamW = Adam + **权重衰减**（Weight Decay）：

```js
// AdamW：带动量自适应 + 权重衰减的梯度下降
weight = weight - learningRate * (gradient + lambda * weight);
//                            ↑                        ↑
//                      梯度本身                额外的 L2 正则化
```

- **Adam**：自动调整每个参数的学习率（大梯度用小学习率，小梯度用大学习率）
- **Weight Decay**：让权重不要太大，防止过拟合

本项目用 `betas=(0.9, 0.95)`：
- 0.9 = 一阶动量（梯度方向）的衰减率
- 0.95 = 二阶动量（梯度平方）的衰减率
