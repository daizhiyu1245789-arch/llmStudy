PyTorch = 深度学习版的 React，nn.Module 组件 = React 组件。

### 2.1 PyTorch 核心概念

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

# ===== nn.Module = React Component =====
class MyModel(nn.Module):
    def __init__(self):
        super().__init__()    # 先调用父类初始化（必写！）
        self.linear = nn.Linear(128, 256)  # 类似于 React 的 state（固定的结构）
        self.dropout = nn.Dropout(0.1)

    def forward(self, x):     # forward = render，组件的核心逻辑
        x = self.linear(x)    # Linear 变换: [b, t, 128] → [b, t, 256]
        x = F.relu(x)         # 激活函数
        x = self.dropout(x)   # Dropout
        return x

model = MyModel()             # 创建实例
output = model(input_tensor)  # 调用 = forward()，等价于 React 的 <MyModel />

# ===== 参数（Parameters）=====
# model.parameters() 返回所有可学习参数，类似 React 的 useEffect 依赖数组
for param in model.parameters():
    print(param.shape)  # torch.Size([256, 128]), torch.Size([256])

# ===== GPU 加速 =====
device = torch.device("cuda" if torch.cuda.is_available() else "cpu")
model = model.to(device)           # 把模型移到 GPU
input_tensor = input_tensor.to(device)  # 把输入也移过去
output = model(input_tensor)       # 在 GPU 上算

# ===== GPU 的类比 =====
# JS 的 Worker/Web Worker = 把计算从主线程移到后台线程
# PyTorch 的 CUDA = 把计算从 CPU 移到 GPU
# GPU 有上千个核心，适合并行做大量简单运算（矩阵乘法是强项）
```

### 2.2 训练一个最简单的模型

```python
import torch
import torch.nn as nn

# ===== 数据 =====（类似 knex.js 查询数据库返回的训练数据）
X = torch.randn(100, 10)    # 100个样本，每个10维
y = torch.randn(100, 1)    # 100个标签，每个1维

# ===== 模型 =====
class LinearRegression(nn.Module):
    def __init__(self):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(10, 32),
            nn.ReLU(),
            nn.Linear(32, 1)
        )

    def forward(self, x):
        return self.net(x)

model = LinearRegression()
loss_fn = nn.MSELoss()           # 均方误差 = 预测值和真实值的差的平方的平均
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)

# ===== 训练循环 =====
for step in range(100):
    pred = model(X)                      # 前向传播
    loss = loss_fn(pred, y)             # 算损失

    optimizer.zero_grad()                # 清零梯度（类似 this.setState({}) 重置）
    loss.backward()                      # 反向传播（算梯度）
    optimizer.step()                     # 更新参数（类似 this.setState(newState)）

    if step % 20 == 0:
        print(f"Step {step}, loss = {loss.item():.4f}")
```

**对比 Node.js 训练循环**：

```js
// Node.js 风格的"训练循环"伪代码
let params = initialParams;

for (let step = 0; step < 100; step++) {
  const pred = forward(input, params);      // 前向传播
  const loss = computeLoss(pred, target);    // 算损失
  const gradient = backprop(loss, params);   // 反向传播
  params = params - learningRate * gradient; // 更新参数
}
```

### 4.4 第四步：部署成本地 API

```python
# server.py
from fastapi import FastAPI
from tinyllm.model import GPT, GPTConfig
from tinyllm.checkpoint import load_checkpoint
from tinyllm.tokenizer import tokenizer_from_extra
import torch

app = FastAPI()

ckpt = load_checkpoint("checkpoints/latest.pt", map_location="cpu")
tokenizer = tokenizer_from_extra(ckpt.get("extra"))
model = GPT(GPTConfig(**ckpt["model_cfg"]))
model.load_state_dict(ckpt["model_state"], strict=True)
model.eval()

@app.post("/generate")
def generate(prompt: str, max_tokens: int = 100, temperature: float = 1.0, top_k: int = 50):
    ids = tokenizer.encode(prompt)
    if not ids:
        ids = [0]
    idx = torch.tensor([ids], dtype=torch.long)
    out = model.generate(idx, max_new_tokens=max_tokens, temperature=temperature, top_k=top_k)
    return {"text": tokenizer.decode(out[0].tolist())}
```

## 常见问题 FAQ

**Q: 我数学不好，能学会吗？**

A: 能。深度学习训练只需要：加、减、乘、除、指数、log。矩阵乘法本质上就是一堆乘法加法。本项目的文档里不出现任何矩阵求导——那是研究人员的事，不是工程师的事。

**Q: 需要买 GPU 吗？**

A: 不需要。本项目是最小配置，CPU 上几分钟就能跑完训练演示。如果以后你想微调真正的大模型（几十亿参数的），那时候再考虑 GPU。Google Colab 提供免费 GPU 可以用。

**Q: 学多久能自己改这个项目？**

A: 按这个路线走：
- Python 基础：1-3 天
- PyTorch 入门：3-7 天
- 概念理解：7-14 天
- 能改代码：再动手 3-5 天
总计约 2-4 周可以有实质性产出。

**Q: 能不能不学 Python，直接用 JavaScript 做深度学习？**

A: 可以，但受限。
- **TensorFlow.js**：能在浏览器/Node.js 里跑模型，但训练不了大模型
- **本项目**：纯 Python + PyTorch，没法直接用 JS 改
长远来看，Python 是深度学习的事实标准，建议还是学一下——语法跟 JS 太像了，1-2 周就能过语法关。
