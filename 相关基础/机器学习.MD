# 小白入门：深度学习是什么？（前端工程师 Node.js 版）

> 如果你能理解 Node.js 的 Event Loop、回调函数和异步流水线的概念，那你就能理解深度学习。
> 因为**深度学习本质上就是一个超级大型的、会自动调整的、用于做预测的函数流水线**。

---

## 1. 先用一个你熟悉的例子理解"机器学习"

### Node.js 里的"函数" vs 深度学习里的"模型"

在 Node.js 里，你写一个函数：

```js
// 给定输入，返回输出
function process(userId) {
  return getUser(userId).then(user => user.name);
}
process(123); // → "张三"
```

在深度学习里，**模型就是一个超级复杂的函数**：

```python
# 给定输入（文字），返回输出（下一个文字的概率）
model(text) → { "今天": 0.3, "天气": 0.2, "你好": 0.1, ... }
```

**区别在哪里？**

| | Node.js 函数 | 深度学习模型 |
|---|---|---|
| 参数是谁写的？ | **你**（手写代码） | **自动从数据中学习** |
| 输入输出固定吗？ | 固定（你定义的） | 不固定（可以处理任意文字） |
| 能处理模糊问题吗？ | 不能（需要精确逻辑） | 能（比如"这段话是正面还是负面？"） |

## 2. 什么是"训练"？—— 就像 npm install 装依赖

想象你写了一个 Node.js 项目，`package.json` 里的依赖版本写得不对，
运行 `node server.js` 的时候程序各种报错。

**训练**就是自动调整这些"依赖版本"的过程：

```
初始状态：package.json 里的依赖版本全是乱的
     ↓
自动跑一遍测试（喂数据）
     ↓
看看结果错得多离谱（计算损失）
     ↓
自动调整每个依赖的版本（反向传播）
     ↓
再跑一遍测试
     ↓
循环，直到结果足够好
```

### 训练三步走（类比 Node.js 启动流程）

```js
// Node.js 启动一个服务
const app = require('./app');     // 1. 加载代码（初始化模型）
const config = require('./config'); // 2. 读取配置（设置超参数）
app.listen(3000);                  // 3. 启动服务（开始训练循环）

// 深度学习训练
model = GPT(cfg);                  // 1. 创建模型结构（定义"函数长什么样"）
optimizer = AdamW(model.params);    // 2. 创建优化器（定义"怎么调整参数"）
for (step in steps) {              // 3. 训练循环
  loss = model(x, y);              //   预测 + 计算错误
  loss.backward();                 //   反向传播（算梯度）
  optimizer.step();                //   更新参数
}
```

## 3. 什么是"神经网络"？—— 就像 Express 中间件链

Express 的中间件链你很熟吧？

```js
app.use(middleware1);  // 处理请求
app.use(middleware2);  // 处理请求
app.use(middleware3);  // 处理响应
```

神经网络就是**很多很多层"中间件"串在一起**，只不过每层都在做数学运算：

```
输入文字
  ↓
Embedding 层（把文字变成数字向量）→ 中间件1
  ↓
Transformer Block 1（注意力计算）→ 中间件2
  ↓
Transformer Block 2（注意力计算）→ 中间件3
  ↓
Transformer Block 3（注意力计算）→ 中间件4
  ↓
Transformer Block 4（注意力计算）→ 中间件5
  ↓
输出层（把向量变回文字概率）→ 最终响应
```

**每层"中间件"都有一些参数（权重）**，训练的过程就是：
自动调整每一层中间件的参数，让最终输出的结果越来越好。

---

## 4. 什么是"参数"？—— 就像数据库的索引调优

在 MongoDB 里，你建了索引，查询就快；没建或建错了，查询就慢。
索引的参数（比如字段权重）是需要根据实际情况调整的。

神经网络里的**参数（权重）**也是一样的道理：
- 参数决定了输入如何变成输出
- 参数不对，输出就错
- 参数需要通过"测试"来自动调整

```
MongoDB:  索引字段选得不对  → 查询慢
神经网络:  权重值设置得不对  → 预测错
```

**区别**：MongoDB 的索引你得手动调，而神经网络的权重是**自动从数据中学习**的。

---

## 5. 什么是"损失函数"？—— 就像单元测试的覆盖率

Node.js 写完代码，跑一下测试：

```js
// 测试结果 vs 期望结果
expect(result).toBe("张三");
```

如果测试没通过，`expect` 会告诉你"错了多少"。

深度学习里的**损失函数（Loss Function）**就是那个 `expect`：

```python
# 模型的预测结果
predictions = model(input_text)

# 用交叉熵损失函数衡量"错得多离谱"
loss = F.cross_entropy(predictions, expected_answer)
```

损失越大 = 错得越离谱，损失越小 = 越接近正确答案。

---

## 6. 什么是"反向传播"？—— 就像 Bug 定位流程

Node.js 生产环境出了 Bug，你的排查流程大概是：

```
第3步出错了 ← 第2步传的数据有问题 ← 第1步算的不对
  ↓
定位到是第1步的问题
  ↓
修复第1步
```

深度学习的**反向传播（Backpropagation）**就是同样的流程：

```
最终输出错了（损失大）
  ↓
是倒数第2层参数的问题
  ↓
是倒数第3层参数的问题
  ↓
一直追溯到第1层
  ↓
把每一层的参数都微调一点点
```

这个过程是**自动完成**的，你不需要手动定位是哪一层出了问题——`loss.backward()` 自动帮你算出来每个参数该往哪个方向调整。

## 7. 什么是"梯度下降"？—— 就像爬山找最低点

想象你在山顶（损失最高），想下山到山谷（损失最低）：

```
山顶（损失最高）
    ⬇️ 往最陡的下坡方向走一步
    ⬇️ 再走一步
    ⬇️ 再走一步
山谷（损失最低）
```

**梯度下降（Gradient Descent）**就是"找到最陡下坡方向"的算法：
- **梯度** = 告诉你在当前位置往哪个方向走最快下山
- **学习率（Learning Rate）** = 每一步迈多大

```
学习率太大：一下迈过山谷，在两个山腰之间来回跳（震荡不收敛）
学习率太小：一步挪一毫米，下辈子都到不了山谷（训练太慢）
```

本项目的默认学习率是 `3e-4`（0.0003），这是一个经过大量实验验证的"不太大也不太小"的值。

## 8. 什么是"Embedding"？—— 就像 Vuex 的 state 映射

在 Vuex 里：

```js
// 原始数据
{ userId: 1, name: "张三", age: 28 }

// 经过 mapState 映射后，组件可以直接用
computed: {
  ...mapState(['userId', 'name', 'age'])
}
```

Embedding 的作用类似，**把离散的文字/字符映射为连续的向量**：

```
原始输入:  "你"   →  token ID: 5
Embedding: 把 5 这个数字 → 变成 [0.23, -1.45, 0.87, ...] 一个 128 维的向量
```

为什么不能直接用数字 5，而要变成向量？
- 直接用数字 5，意味着"你"和"好"的距离是 `6-5=1`——但语言里"你"和"好"不一定比"你"和"我"更相关
- Embedding 向量可以表达语义关系：相近语义的词，在向量空间里距离更近

## 9. 什么是"注意力机制"？—— 就像事件总线里的消息订阅

Node.js 的 EventEmitter 你熟吧？

```js
eventBus.on('user:login', (data) => { /* 处理 */ });
eventBus.on('user:login', (data) => { /* 另一个处理 */ });
```

**注意力机制（Attention）**的思路类似：
对于序列中的每个位置，模型需要"关注"序列中其他位置的信息，来更好地理解当前位。

```
输入序列:  [今, 天, 天, 气, 很, 好]
               ↓
Attention 计算：
  - 处理"气"的时候，需要看看"今""天""天"来理解上下文
  - 处理"好"的时候，需要看看"很"来理解是"很好"还是"不好"
               ↓
每个位置都有一个"关注权重"，决定它应该多关注序列中的哪些位置
```

**自注意力（Self-Attention）**就是：输入序列自己关注自己，Q（查询）、K（键）、V（值）都来自同一个序列。

## 10. 什么是"Transformer"？—— 就像微服务架构里的 API 网关

Transformer 是本项目的核心架构。它由多个组件组成：

```
输入 Embedding（解析请求）→ 位置编码（附加请求路径信息）
        ↓
    Transformer Block × N（多层处理）
        ↓
  输出投影 + Softmax（格式化响应）
        ↓
  生成下一个 token（继续调用 API）
```

| Transformer 组件 | 类比 Node.js |
|---|---|
| Token Embedding | 请求体的 JSON.parse（把文本解析成结构化数据） |
| Position Embedding | 请求路径路由（告诉模型 token 在序列的哪个位置） |
| LayerNorm | 批处理标准化（就像 lodash 的 _.flatten 规范化数据） |
| MLP（前馈网络） | 中间件的业务逻辑处理 |
| 因果掩码 | 防止看到"未来"的 token（类似 CORS 只允许特定来源） |

## 11. 什么是"Tokenizer"？—— 就像 JSON.parse 的反义词

```
JSON.parse:   '{"name": "张三"}'  →  JavaScript 对象
JSON.stringify: JavaScript 对象  →  '{"name": "张三"}'

Tokenize:    "今天天气很好"  →  [5, 23, 42, 88, 102, 15]  (文字 → ID)
Detokenize:  [5, 23, 42, 88, 102, 15]  →  "今天天气很好"  (ID → 文字)
```

本项目有两种分词器：

| | CharTokenizer（字符级） | ByteTokenizer（字节级） |
|---|---|---|
| 切分方式 | 每个字符一个 ID | 每个字节（0-255）一个 ID |
| 例子 | "你" → [5], "好" → [6] | "你"（UTF-8: `0xE4 0xBD 0xA0`）→ [228, 189, 160] |
| 词表大小 | 取决于语料中出现了哪些字符 | 固定 256 |
| 优点 | 序列短，可读 | 可处理任意文本 |

## 12. 什么是"GPT"？—— 就是这个项目做的事

GPT = **G**enerative **P**re-trained **T**ransformer

- **Generative（生成式）**：给定前文，预测下一个词（自回归生成）
- **Pre-trained（预训练）**：在大规模语料上先学一遍"语言规律"，再微调
- **Transformer**：使用 Transformer 架构

```
# GPT 生成的过程
输入: "今天"
模型说: 下一个字有 30% 概率是"天"，20% 是"好"，10% 是"的"...
    ↓
随机抽一个（或者按概率抽）
    ↓
输出: "今天" + "天" = "今天天"
    ↓
再把"今天天"输入模型，预测下一个字...
    ↓
循环 max_new_tokens 次
    ↓
最终输出: "今天天气很不错呀"
```

## 13. 这个 tinyllm 项目是怎么跑起来的？

```
1. 准备语料（文本文件）
       ↓
2. Tokenize（把文字变成数字 ID）
       ↓
3. 创建模型（GPT 架构）
       ↓
4. 训练循环（几千步的 预测→算损失→反向传播→更新参数）
       ↓
5. 保存 checkpoint（best.pt / latest.pt）
       ↓
6. 加载 checkpoint，用 prompt 生成新文本
```

训练一次（在 CPU 上）大概 10-30 分钟（取决于数据量和参数）。
生成一次（推理）几秒钟搞定。

## 14. 常见名词速查表

| 名词 | 一句话解释 | Node.js 类比 |
|------|-----------|-------------|
| 模型（Model） | 一个超级复杂的函数 | 一个写好的 Node.js API |
| 参数（Parameter） | 模型内部的值，通过训练学会 | 数据库索引的具体配置 |
| 损失函数（Loss） | 衡量预测错得多离谱 | 单元测试的失败数 |
| 反向传播（Backprop） | 自动找出每个参数该调多少 | Bug 追踪定位过程 |
| 梯度下降（Gradient Descent） | 找到最陡下山方向 | 爬山找山谷最低点 |
| 学习率（Learning Rate） | 每步调整的幅度 | 爬山时每步迈多远 |
| Embedding | 把文字变成数字向量 | JSON.parse |
| Attention | 决定每个位置该关注哪些其他位置 | EventBus 消息订阅 |
| Transformer | 很多 Attention + MLP 堆叠的结构 | Express 中间件链 |
| Tokenizer | 把文字切成小片段的工具 | URL 路由解析 |
| GPT | 用 Transformer 做语言生成的模型 | 一个生成 RESTful 响应的服务 |
| Batch Size | 一次喂给模型多少条样本 | 批量插入数据库 |
| Epoch | 把整个语料过一遍 | 遍历一整棵 DOM 树 |
