
## 1. 模型基础

### AI

人工智能。很大的概念，泛指让机器表现出智能行为。

### Machine Learning

机器学习。让模型从数据里学习规律，而不是人手写所有规则。

### Deep Learning

深度学习。机器学习的一类，核心是多层神经网络。

### LLM

Large Language Model，大语言模型。

例如 GPT、LLaMA、Qwen 这类模型。

### Multimodal Model

多模态模型。能同时处理文字、图片、音频、视频等多种输入。

### Parameter

参数。模型训练出来的权重。

直觉：

```text
参数 = 模型记住和表达知识的数字
```

### Model Size

模型规模，通常用参数量表示，比如 7B、13B、70B。

```text
7B = 70 亿参数
```

## 2. 文本和 Token

### Token

模型处理文本的基本单位。

可能是：

- 一个字
- 一个词
- 一个子词
- 一个符号

### Tokenizer

分词器。负责：

```text
文本 → token ID
token ID → 文本
```

### Vocabulary

词表。模型认识的所有 token 集合。

### Token ID

token 在词表里的编号。

例如：

```text
"今天" → [123, 456]
```

### Context Window

上下文窗口。模型一次最多能看的 token 数。

上下文越长，能塞的历史对话和资料越多，但成本也更高。

---

## 3. 向量和 Embedding

### Vector

向量，一串数字。

在大模型里，它常常表示一个 token、一句话或一段文档的语义。

### Embedding

把文本变成向量。

```text
"Node 怎么接 Hugging Face" → [0.12, -0.33, ...]
```

### Semantic Similarity

语义相似度。判断两段文本意思是否接近。

### Cosine Similarity

余弦相似度。常用于比较两个 embedding 是否接近。

### Vector Database

向量数据库。专门存 embedding，并支持相似度搜索。

常见：

- pgvector
- Qdrant
- Milvus
- Weaviate
- Pinecone

---

## 4. 神经网络

### Tensor

张量。多维数组。

PyTorch 里几乎所有数据都是 tensor。

### Layer

层。神经网络中的一个计算模块。

### Linear Layer

线性层。

```text
y = xW + b
```

### Activation

激活函数。给网络引入非线性。

常见：

- ReLU
- GELU

### MLP

多层感知机。Transformer 里的前馈网络通常就是 MLP。

### LayerNorm

层归一化，让数值更稳定，训练更容易。

### Dropout

训练时随机丢弃部分神经元，减少过拟合。

---

## 5. Transformer 和 Attention

### Transformer

现在大模型最核心的网络架构。

### Attention

注意力机制。让模型决定当前 token 应该重点看上下文里的哪些 token。

### Self-Attention

自注意力。同一段文本内部互相看。

### Causal Attention

因果注意力。只能看当前位置之前的 token，不能偷看未来。

GPT 就是因果语言模型。

### Q / K / V

Attention 里的三个向量：

```text
Q = Query，当前想找什么
K = Key，别人提供什么索引
V = Value，别人实际携带的信息
```

### Multi-Head Attention

多头注意力。多个注意力头并行看不同关系。

### Positional Embedding

位置嵌入。告诉模型 token 在序列里的位置。

---

## 6. 训练

### Training

训练。用数据更新模型参数。

### Inference

推理。用训练好的模型生成结果。

### Loss

损失。模型错了多少。

### Cross Entropy

交叉熵。分类和语言模型常用的 loss。

### Gradient

梯度。告诉参数往哪个方向改，loss 会下降。

### Backpropagation

反向传播。自动计算每个参数的梯度。

### Optimizer

优化器。根据梯度更新参数。

常见：

- SGD
- Adam
- AdamW

### Learning Rate

学习率。每次参数更新走多大一步。

### Batch

批次。一次训练喂给模型的一组样本。

### Epoch

完整看完一遍训练数据。

### Checkpoint

模型存档。保存训练到某一步的模型参数和状态。

---

## 7. 生成

### Logits

模型输出的原始分数，还不是概率。

### Softmax

把 logits 转成概率分布。

### Temperature

控制生成随机性。

```text
低 temperature：稳定保守
高 temperature：随机发散
```

### Top-k

只从概率最高的 k 个 token 里选。

### Top-p

也叫 nucleus sampling。只从累计概率达到 p 的候选里选。

### Max New Tokens

最多生成多少个新 token。

---

## 8. Prompt 和应用

### Prompt

给模型的输入指令。

### System Prompt

系统提示词。定义模型角色、规则、输出边界。

### Prompt Template

Prompt 模板。把固定规则和变量组合起来。

### Structured Output

结构化输出，比如 JSON。

### Prompt Injection

提示词注入。用户输入试图覆盖你的系统规则。

---

## 9. RAG

### RAG

Retrieval-Augmented Generation，检索增强生成。

```text
先检索资料，再让模型基于资料回答。
```

### Chunk

文档切块。把长文档拆成多个小片段。

### Retrieval

检索。从知识库里找到和问题相关的资料。

### TopK

检索最相关的 K 条结果。

### Source Citation

引用来源。回答里标注答案来自哪个文档片段。

---

## 10. Tool Calling 和 Agent

### Tool Calling

让模型选择调用后端工具。

例如：

```text
getWeather({ city: "杭州" })
```

### Tool Schema

描述工具名、用途、参数格式。

### Agent

工程里常说的 Agent 通常是：

```text
大模型 + 工具调用 + 状态 + 执行循环
```

### Planning

规划。模型把任务拆成多个步骤。

### Memory

记忆。保存历史信息，供后续任务使用。

## 11. 部署和工程

### API Key

调用模型服务的密钥。

不要放前端。

### Environment Variable

环境变量。运行时配置，比如 `HF_TOKEN`。

### SSE

Server-Sent Events。服务端持续向浏览器推送文本事件，常用于流式输出。

### Rate Limit

限流。限制用户请求频率。

### Cache

缓存。减少重复请求和成本。

### Evaluation

评估。用固定问题检查模型效果。

### Observability

可观测性。日志、指标、链路追踪，让你知道系统哪里慢、哪里错。

## 12. 最容易混淆的几组词

### Tokenizer vs Embedding

```text
Tokenizer：文本和 token ID 互转
Embedding：文本或 token 变成语义向量
```

### Training vs Inference

```text
Training：训练模型，更新参数
Inference：使用模型，不更新参数
```

### RAG vs Fine-tuning

```text
RAG：给模型外部资料
Fine-tuning：继续训练模型参数
```

知识频繁更新，优先 RAG。

### Agent vs Tool Calling

```text
Tool Calling：模型调用一个工具
Agent：模型多步骤选择、调用工具、观察结果、继续执行
```

### Prompt vs Context

```text
Prompt：完整输入指令
Context：prompt 里的参考资料或历史上下文
```
