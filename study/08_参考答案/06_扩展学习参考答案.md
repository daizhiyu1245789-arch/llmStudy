## 01_必读论文

### Attention is All You Need

参考总结：

```text
论文名：Attention is All You Need
它解决什么问题：RNN 难并行、长距离依赖不好学
核心方法：完全基于 Attention 的 Transformer
和 tinyllm 的关系：tinyllm 的 Attention、Position Embedding、Block 都来自 Transformer 思路
我还没看懂的地方：可以写任何你不懂的公式或实验细节
```

### GPT-2

参考总结：

```text
论文名：Language Models are Unsupervised Multitask Learners
它解决什么问题：证明大语言模型通过预训练和 prompt 可以做多任务
核心方法：GPT 风格因果语言模型，大规模 WebText 训练
和 tinyllm 的关系：tinyllm 是一个极小版 GPT 风格模型
```

### BERT vs GPT

```text
BERT：双向上下文，适合理解任务
GPT：从左到右，适合生成任务
```

## 02_Hugging Face 入门

### 模型卡记录模板示例

```text
模型名：gpt2
任务类型：text-generation
支持语言：主要英文
如何调用：Transformers / Inference API / @huggingface/inference
License：查看 model card
适合我的项目吗：适合测试链路，不适合中文生产问答
```

### Hugging Face 关键概念

```text
Hub：模型、数据集、demo 的托管平台
Model：训练好的模型权重和配置
Tokenizer：模型配套分词器
Dataset：数据集
Transformers：本地加载和运行模型的库
Inference API：通过 API 调用模型服务
```

### 找模型时重点看

- task
- language
- license
- model size
- example usage
- limitations
- 是否还在维护

## 03_Node 开发者大模型学习路线

### Node 开发者优势参考答案

```text
Node 开发者熟悉 HTTP API、异步、流式响应、前后端联调、npm 生态。
这些能力正好对应大模型应用里的接口封装、SSE、RAG、工具调用和部署。
```

### SSE 为什么重要？

```text
大模型生成通常比较慢，SSE 可以让模型边生成边返回，用户体验更好。
```

### Prompt 为什么像接口设计？

```text
Prompt 定义输入字段、任务规则、上下文和输出格式。
写得越像协议，模型输出越稳定。
```

### RAG 完整链路

```text
文档 → 切块 → embedding → 向量库
用户问题 → embedding → 检索 topK
topK chunks → prompt → 模型回答 → 引用来源
```

### Tool Calling 和 Agent

```text
Tool Calling：模型选择调用某个工具。
Agent：模型在多步骤任务里选择工具、观察结果、继续执行。
```

### 最终项目设计示例

```text
项目名：个人知识库问答助手
目标用户：想管理自己学习资料的开发者
核心功能：上传 Markdown，基于资料问答，回答带引用
使用模型：Hugging Face 文本生成模型
是否需要 RAG：需要
需要哪些工具调用：searchDocs、getCurrentTime、summarizeDoc
怎么部署：Node API + 前端静态页面 + Docker
怎么评估：准备 20 个固定问题，检查关键词和引用来源
```

### 最终验收回答

```text
我能用 Node 把模型 API 封装成服务，支持流式输出；

```
