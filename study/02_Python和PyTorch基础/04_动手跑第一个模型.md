# 动手跑第一个模型：从安装到生成文本

> 这一篇手把手带你把 tinyllm 项目跑起来。跑通之后，你就有了第一个"能跑的深度学习模型"。

---

## 目标

```
输入：python -m tinyllm.train ...
输出：训练完成，生成 checkpoint 文件

输入：python -m tinyllm.generate ...
输出：用训练好的模型生成一段文字
```

---

## 步骤一：安装依赖

### 1.1 创建虚拟环境（推荐）

```bash
# 在项目根目录下创建虚拟环境
python -m venv .venv

# Windows 激活虚拟环境
.venv\Scripts\activate

# Linux / Mac 激活
# source .venv/bin/activate
```

虚拟环境类似 Node.js 项目的 `node_modules/` 隔离，每个项目可以有不同的依赖版本。

### 1.2 安装 requirements.txt 里的依赖

```bash
pip install -r requirements.txt
```

本项目的 requirements.txt 内容：
```
torch
numpy
tqdm
```

`torch` 是最大的依赖（几百 MB），安装需要等一会儿。

## 步骤三：跑第一次训练

### 3.1 用默认参数（CPU，小规模）

```bash
python -m tinyllm.train \
    --data_path data/sample.txt \
    --max_steps 50 \
    --log_interval 10 \
    --eval_interval 20 \
    --save_interval 20
```

参数解释：
```
--data_path      训练语料文件路径
--max_steps      训练多少步（50步大约 1-2 分钟 CPU）
--log_interval   每隔多少步打印一次日志
--eval_interval  每隔多少步评估一次验证集
--save_interval  每隔多少步保存一次 checkpoint
```

### 3.2 预期输出

```
100%|██████████| 50/50 [01:23<00:00,  1.67s/step]
step 11 loss 3.2812 (3.2s)
step 21 loss 2.9451 (6.1s)
step 31 loss 2.5123 (9.4s)
step 41 loss 2.2034 (12.8s)
step 51 loss 1.9876 (16.2s)
```

- `loss` 在**下降** = 模型在学习
- `loss` 降到 1.0 左右才能生成比较像样的文字
- 50 步太少，只是演示，跑 200 步会更好

### 3.3 训练完成后检查 checkpoint

```bash
# 查看 checkpoints 目录
ls checkpoints/

# 应该看到：
# best.pt      ← 最佳模型
# latest.pt    ← 最新存档
# step_20.pt   ← 第20步存档
# step_40.pt   ← 第40步存档
```

---

## 步骤四：用训练好的模型生成文本

```bash
python -m tinyllm.generate \
    --ckpt_path checkpoints/latest.pt \
    --prompt "今天" \
    --max_new_tokens 50 \
    --temperature 0.9 \
    --top_k 50
```

参数解释：
```
--ckpt_path      用哪个 checkpoint（latest.pt / best.pt）
--prompt         生成的开头（给模型一个"提示"）
--max_new_tokens 生成多少个新 token（中文约等于多少个汉字）
--temperature    随机性（0.9 = 有点创意但不乱来）
--top_k          只从概率最高的50个token里选
```

**预期输出**：
```
今天天气真是不错，阳光温暖地洒在街道上...
```

（如果 loss 还很高（>2.0），输出可能是乱码或不通顺，这是正常的，训练不够。）

## 步骤五：尝试不同的参数

### 5.1 更大的模型（更慢但更好）

```bash
python -m tinyllm.train \
    --data_path data/sample.txt \
    --n_layer 6 \
    --n_head 6 \
    --n_embd 256 \
    --max_steps 200 \
    --log_interval 20 \
    --eval_interval 50 \
    --save_interval 50
```

| 参数 | 默认值 | 增大后 | 效果 |
|------|--------|--------|------|
| `n_layer` | 4 | 6 | 更深的模型，能学更复杂的关系 |
| `n_head` | 4 | 6 | 更多的注意力头 |
| `n_embd` | 128 | 256 | 更大的向量维度，容量更大 |

### 5.2 更长的上下文

```bash
python -m tinyllm.train \
    --data_path data/sample.txt \
    --block_size 256 \
    --max_steps 200
```

`block_size=256` 意味着模型能看到更长的上下文（256 个 token），生成更连贯的长文本。

### 5.3 字节级分词

```bash
python -m tinyllm.train \
    --data_path data/sample.txt \
    --tokenizer byte \
    --max_steps 200
```

适合处理特殊文本（如代码、日志、混合格式）。

### 5.4 生成时调整随机性

```bash
# 高随机性（更有创意但可能跑题）
python -m tinyllm.generate \
    --ckpt_path checkpoints/latest.pt \
    --prompt "今天" \
    --max_new_tokens 100 \
    --temperature 1.5

# 低随机性（更保守、更确定）
python -m tinyllm.generate \
    --ckpt_path checkpoints/latest.pt \
    --prompt "今天" \
    --max_new_tokens 100 \
    --temperature 0.5

# 贪婪解码（最确定，直接选概率最高的）
python -m tinyllm.generate \
    --ckpt_path checkpoints/latest.pt \
    --prompt "今天" \
    --max_new_tokens 100 \
    --temperature 0
```

## 常见问题排查

### 问题 1：torch 安装失败

```bash
# 如果 torch 安装慢，试一下国内镜像
pip install torch -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### 问题 2：loss 一直是 3.x 不下降

正常，50 步太少了，模型还没开始学习。`loss` 从 ~3.3 降到 ~2.0 需要约 100-200 步，降到 ~1.0 需要 500+ 步。

### 问题 3：生成的文字是乱码

1. 训练步数不够（loss 还太高）
2. 尝试 `temperature 0.7` 不要太高
3. 检查 checkpoint 是否正确加载

### 问题 4：报内存不足（OOM）

```bash
# 减小 batch_size
python -m tinyllm.train \
    --data_path data/sample.txt \
    --batch_size 8    # 默认是 32，显存不够时减小

# 或者减小模型
python -m tinyllm.train \
    --data_path data/sample.txt \
    --n_embd 64 \
    --n_layer 2
```

### 问题 5：训练中断后继续

```bash
python -m tinyllm.train \
    --data_path data/sample.txt \
    --resume \
    --max_steps 200
```

`--resume` 会从 `checkpoints/latest.pt` 继续训练，不从头开始。
