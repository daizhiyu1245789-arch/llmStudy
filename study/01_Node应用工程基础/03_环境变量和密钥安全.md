# 环境变量和密钥安全

> 大模型应用几乎一定会接 API Key，比如 Hugging Face Token。密钥管理没做好，项目还没上线就先漏了。

---

## 0. 学完这一篇你会什么？

你会学会：

- `.env` 是什么
- 为什么密钥不能写进代码
- 为什么密钥不能放前端
- Node 里怎么读取环境变量
- 怎么写 `.env.example`
- 怎么避免把 `.env` 提交到 Git
- 大模型应用里哪些信息算敏感

---

## 1. 什么是环境变量？

环境变量就是运行程序时外部传入的配置。

例如：

```bash
HF_TOKEN=hf_xxx
HF_MODEL=gpt2
PORT=3001
```

代码里读取：

```ts
const token = process.env.HF_TOKEN;
```

好处：

- 本地、测试、生产可以用不同配置
- 密钥不需要写死在代码里
- 部署平台可以统一管理配置

---

## 2. 为什么不能把密钥写进代码？

不要这样：

```ts
const HF_TOKEN = "hf_xxxxxxxxx";
```

风险：

- Git 历史会永久保存
- 代码发给别人就泄露
- 前端打包会暴露
- 以后轮换密钥麻烦

应该这样：

```ts
const HF_TOKEN = process.env.HF_TOKEN;
```

---

## 3. `.env` 和 `.env.example`

`.env`：本地真实配置。

```bash
HF_TOKEN=hf_your_real_token
HF_MODEL=gpt2
PORT=3001
```

`.env.example`：给别人看的模板。

```bash
HF_TOKEN=hf_your_token_here
HF_MODEL=gpt2
PORT=3001
```

`.env.example` 可以提交 Git。

`.env` 不要提交 Git。

---

## 4. `.gitignore`

在项目里加：

```text
.env
node_modules/
dist/
```

如果 `.env` 已经被提交过，光加 `.gitignore` 不够，需要从 Git 历史处理并立即轮换密钥。

入门阶段先记住：

```text
先写 .gitignore，再创建 .env。
```

---

## 5. Node 里读取 `.env`

安装：

```bash
npm install dotenv
```

入口文件：

```ts
import "dotenv/config";
```

封装配置：

```ts
function optionalEnv(name: string): string | undefined {
  const value = process.env[name]?.trim();
  return value ? value : undefined;
}

export const config = {
  port: Number(optionalEnv("PORT") ?? 3001),
  hfToken: optionalEnv("HF_TOKEN"),
  hfModel: optionalEnv("HF_MODEL") ?? "gpt2",
};
```

必填配置要检查：

```ts
if (!config.hfToken) {
  throw new Error("Missing HF_TOKEN");
}
```

---

## 6. 为什么 token 不能放前端？

前端代码会发给用户浏览器。

即使你写：

```js
const token = "hf_xxx";
```

或者：

```js
import.meta.env.VITE_HF_TOKEN
```

只要打包到前端，用户就可能看到。

正确结构：

```text
浏览器
  ↓ 调自己的 Node API
Node 后端
  ↓ 带 HF_TOKEN 调 Hugging Face
Hugging Face
```

---

## 7. 大模型应用里的敏感信息

这些都要小心：

- Hugging Face Token
- OpenAI / Anthropic / 其他模型 API Key
- 数据库密码
- 用户邮箱、手机号
- 用户上传的私有文档
- RAG 检索出来的内部资料
- 工具调用结果
- Prompt 中的内部规则
- 日志里的用户问题

日志也不要乱记敏感信息。

---

## 8. 生产环境怎么配置？

本地：

```bash
.env
```

Docker：

```bash
docker run -e HF_TOKEN=hf_xxx app
```

云平台：

```text
在平台环境变量页面配置 HF_TOKEN
```

不要把真实密钥写进 Dockerfile：

```dockerfile
# 不要这样
ENV HF_TOKEN=hf_xxx
```
