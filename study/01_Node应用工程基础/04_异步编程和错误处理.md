## 1. async / await 基础

异步函数返回 Promise：

```ts
async function generateText(prompt: string): Promise<string> {
  const result = await callModel(prompt);
  return result.text;
}
```

调用：

```ts
const text = await generateText("hello");
```

如果出错，会抛异常：

```ts
try {
  const text = await generateText(prompt);
} catch (error) {
  console.error(error);
}
```

## 2. 大模型应用里常见异步操作

| 操作 | 为什么异步 |
|---|---|
| 调 Hugging Face | 网络请求 |
| 生成 embedding | 网络请求或模型计算 |
| 读用户上传文件 | 文件 IO |
| 查向量库 | 数据库 IO |
| 工具调用 | 外部 API / 数据库 |
| SSE 输出 | 长连接 |

所以后端接口几乎都会是：

```ts
app.post("/api/xxx", async (req, res, next) => {
  try {
    // await ...
  } catch (error) {
    next(error);
  }
});
```

## 3. Express 统一错误处理

接口里：

```ts
app.post("/api/generate", async (req, res, next) => {
  try {
    const result = await generateText(req.body.prompt);
    res.json({ result });
  } catch (error) {
    next(error);
  }
});
```

统一错误中间件：

```ts
app.use((error: unknown, req, res, next) => {
  console.error(error);

  res.status(500).json({
    error: "Internal server error",
  });
});
```

如果是 Zod 参数错误：

```ts
if (error instanceof z.ZodError) {
  res.status(400).json({
    error: "Invalid request",
    issues: error.issues,
  });
  return;
}
```

## 4. 错误信息不要全部暴露给前端

不要直接：

```ts
res.status(500).json({ error: String(error) });
```

因为错误里可能有：

- 内部路径
- 请求参数
- 密钥片段
- 数据库信息
- 第三方接口细节

更好的做法：

```ts
console.error({ requestId, error });

res.status(500).json({
  requestId,
  error: "模型服务暂时不可用，请稍后重试",
});
```

## 5. 超时控制

模型请求可能很慢。你需要超时。

通用工具：

```ts
export function withTimeout<T>(promise: Promise<T>, ms: number): Promise<T> {
  return new Promise((resolve, reject) => {
    const timer = setTimeout(() => {
      reject(new Error(`Timeout after ${ms}ms`));
    }, ms);

    promise
      .then(resolve)
      .catch(reject)
      .finally(() => clearTimeout(timer));
  });
}
```

使用：

```ts
const text = await withTimeout(generateText(input), 60_000);
```

## 6. 哪些错误可以重试？

可以重试：

- 网络临时失败
- 第三方服务 502 / 503
- 请求超时
- 模型输出 JSON 格式坏了，可以让模型修复一次

不要重试：

- 参数校验失败
- token 无效
- 权限不足
- 用户输入过长
- 余额不足

重试也要有限制：

```text
最多 1-2 次
指数退避
记录重试次数
```

## 7. SSE 中途出错

如果还没发响应头，可以正常返回错误。

如果已经开始流式输出，只能写 SSE 错误事件：

```ts
if (res.headersSent) {
  res.write(`data: ${JSON.stringify({ error: "模型调用失败" })}\n\n`);
  res.end();
  return;
}
```

前端：

```js
if (data.error) {
  throw new Error(data.error);
}
```

## 8. requestId

每个请求生成一个 ID：

```ts
import crypto from "node:crypto";

export function createRequestId() {
  return crypto.randomUUID();
}
```

日志和响应都带上：

```ts
console.error({ requestId, error });

res.status(500).json({
  requestId,
  error: "请求失败",
});
```

这样用户反馈问题时，你能查到对应日志。
