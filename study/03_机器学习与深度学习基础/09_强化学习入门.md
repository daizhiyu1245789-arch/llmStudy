# 强化学习入门

> 强化学习不是“给标准答案学”，而是让智能体通过试错和奖励学会做决策。

---

## 1. 强化学习解决什么问题？

监督学习的数据长这样：

```
输入 → 正确答案
```

强化学习的数据更像这样：

```
当前状态 → 做一个动作 → 环境反馈奖励 → 进入下一个状态
```

例子：

| 场景 | 状态 | 动作 | 奖励 |
|---|---|---|---|
| 游戏 | 当前画面 | 上下左右/攻击 | 得分、通关 |
| 机器人 | 传感器数据 | 电机控制 | 走得稳、不摔倒 |
| 推荐系统 | 用户上下文 | 推荐内容 | 点击、停留、购买 |
| 对话模型 | 当前对话 | 生成回答 | 人类偏好评分 |

强化学习适合“连续决策”问题：当前选择会影响后面的局面。

---

## 2. 强化学习的五个核心概念

### 2.1 Agent：智能体

做决策的人。

```
游戏里的玩家
机器人控制器
推荐系统策略
对话模型
```

### 2.2 Environment：环境

智能体互动的世界。

```
游戏关卡
真实物理世界
用户系统
对话评价系统
```

### 2.3 State：状态

当前局面。

```
棋盘布局
机器人姿态
用户画像
当前对话上下文
```

### 2.4 Action：动作

智能体可以做的选择。

```
向左走
选择一个商品推荐
生成下一句话
调用一个工具
```

### 2.5 Reward：奖励

环境给智能体的反馈。

```
赢了 +1
输了 -1
用户点击 +1
回答更符合偏好 +1
```

强化学习的目标：

```
学到一个策略 policy，让长期累计奖励最大。
```

---

## 3. 强化学习和监督学习的区别

| | 监督学习 | 强化学习 |
|---|---|---|
| 数据形式 | 输入 + 正确答案 | 状态 + 动作 + 奖励 |
| 学习方式 | 模仿答案 | 试错探索 |
| 目标 | 单次预测准确 | 长期奖励最大 |
| 难点 | 数据质量、泛化 | 探索、延迟奖励、稳定性 |
| 例子 | 图片分类、文本分类 | 游戏、机器人、策略优化 |

监督学习像刷有答案的题。

强化学习像玩游戏：很多时候你不知道当前这一步是不是最好，要过一会儿才知道结果。

---

## 4. 一个最小例子：走迷宫

```
S . . #
. # . .
. . . G
```

- `S`：起点
- `G`：终点
- `#`：墙
- 动作：上、下、左、右
- 奖励：到终点 +10，撞墙 -1，每走一步 -0.1

智能体一开始乱走，后来慢慢学会：

```
哪些状态该往右
哪些状态该往下
哪些地方不能撞墙
```

最后形成一个策略：

```
policy(state) → action
```

## 5. 常见强化学习算法

### 5.1 Q-Learning

学习一个 Q 表：

```
Q(state, action) = 在某个状态下做某个动作，长期看有多好
```

适合：

- 状态和动作数量不大
- 网格世界
- 简单游戏

### 5.2 DQN

用神经网络近似 Q 表。

```
state → 神经网络 → 每个 action 的 Q 值
```

DQN 适合状态空间很大的场景，比如游戏画面。

### 5.3 Policy Gradient

直接学习策略：

```
state → 神经网络 → action 概率
```

如果某次动作带来高奖励，就提高以后做类似动作的概率。

### 5.4 PPO

PPO 是目前很常用的强化学习算法之一，特点是训练相对稳定。

它常用于：

- 游戏智能体
- 机器人控制
- 大模型对齐相关训练

## 6. 探索 vs 利用

强化学习最经典的问题：

```
利用 Exploitation：选择当前看起来最好的动作
探索 Exploration：尝试没试过的动作，可能发现更好策略
```

例子：

```
你已经知道 A 餐厅 80 分
B 餐厅没吃过，可能 50 分，也可能 95 分
今天去哪吃？
```

只利用：永远吃 A，稳定但可能错过 B。

只探索：天天换，学得多但体验不稳定。

好的策略需要平衡。

## 7. 强化学习和大模型有什么关系？

大模型的基础训练通常不是强化学习，而是：

```
预训练：预测下一个 token
指令微调：学习用户指令和理想回答
```

强化学习更多出现在“对齐”阶段。

经典路线：

```
预训练模型
  ↓
指令微调 SFT
  ↓
人类偏好数据
  ↓
奖励模型 Reward Model
  ↓
RLHF / PPO
  ↓
更符合人类偏好的助手模型
```

现在也有很多不用完整 RL 的偏好优化方法，比如 DPO。入门阶段先理解 RLHF 的思想即可：

```
不是只让模型续写文本，而是让模型更倾向生成“人类更喜欢”的回答。
```

## 8. 强化学习和 Agent 的区别

现在很多“大模型 Agent”不一定用了强化学习训练。

工程里的 Agent 通常指：

```
大模型 + 工具调用 + 任务规划 + 记忆/状态 + 执行循环
```

强化学习里的 Agent 指：

```
通过状态、动作、奖励学习策略的智能体
```

两者有交集，但不是一回事。

你做 LLM 应用时，更多会先接触工程 Agent：

- 让模型选择调用哪个工具
- 根据工具结果继续推理
- 多步骤完成任务
- 记录中间状态

---

## 9. 什么时候需要强化学习？

适合考虑强化学习的情况：

- 决策会影响后续状态
- 没有明确标准答案
- 目标是长期收益
- 可以定义奖励
- 可以大量试错或模拟

不适合一上来就用强化学习的情况：

- 普通分类问题
- 简单文本生成
- 数据很少且无法试错
- 奖励很难定义
- 用规则或监督学习已经很好解决
