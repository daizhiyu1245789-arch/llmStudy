# 神经网络进阶

> 前一篇讲了神经元、层、反向传播和优化器。这一篇补充神经网络里更常见、更实战的知识。

---

## 1. 神经网络到底在学什么？

神经网络本质上是在学习一个复杂函数：

```
输入 x → 多层变换 → 输出 y
```

比如：

| 任务 | 输入 | 输出 |
|---|---|---|
| 图片分类 | 一张图片 | 猫 / 狗 / 车 |
| 文本分类 | 一句话 | 正面 / 负面 |
| 语言模型 | 前文 token | 下一个 token 的概率 |
| 回归预测 | 房屋信息 | 房价 |

每一层做的事情可以粗略理解为：

```
底层：学简单特征
中层：组合简单特征
高层：形成任务相关表示
```

在大模型里，Transformer 的很多层也在做类似的事情：低层更偏词法和局部模式，高层更偏语义、任务和推理格式。

## 2. 常见网络结构

### 2.1 MLP：多层感知机

最基础的神经网络，也叫全连接网络。

```
输入 → Linear → 激活 → Linear → 激活 → Linear → 输出
```

适合：

- 表格数据
- 简单分类
- 简单回归
- Transformer 里的 FFN / MLP 子层

本项目的 `MLP` 就是 Transformer Block 里的前馈网络。

### 2.2 CNN：卷积神经网络

常用于图像。

```
图片 → 卷积提取局部特征 → 池化降采样 → 分类
```

直觉：

```
先识别边缘、纹理
再识别眼睛、轮子、形状
最后识别猫、车、人
```

适合：

- 图像分类
- 目标检测
- 医学影像
- OCR 里的视觉部分

### 2.3 RNN / LSTM / GRU

早期常用于序列数据。

```
第 1 个 token → 隐状态
第 2 个 token → 更新隐状态
第 3 个 token → 更新隐状态
...
```

问题：

- 长距离依赖难学
- 训练不能很好并行
- 很长文本容易遗忘前面信息

Transformer 出现后，很多 NLP 任务都转向了 Attention。

### 2.4 Transformer

现在大模型的主流结构。

```
Token Embedding
  ↓
Attention + MLP × N
  ↓
输出
```

优势：

- 能直接关注远距离 token
- 训练并行度高
- 扩大参数和数据后效果非常强

本项目 tinyllm 就是 GPT 风格的因果 Transformer。

## 3. 前向传播里发生了什么？

以分类任务为例：

```
输入数据
  ↓
模型算出 logits
  ↓
softmax 变成概率
  ↓
和真实标签比较
  ↓
得到 loss
```

语言模型也类似，只是输出不是几个类别，而是整个词表的概率：

```
输入 token 序列
  ↓
每个位置输出 vocab_size 个分数
  ↓
预测下一个 token
```

本项目里：

```python
logits, loss = model(x, y)
```

- `logits`：模型对下一个 token 的打分
- `loss`：预测和真实答案之间的差距

## 4. 训练时最常见的问题

### 4.1 欠拟合

模型太弱，训练集都学不好。

表现：

```
train loss 高
val loss 也高
生成结果很乱
```

常见解决：

- 增大模型：更多层、更大 embedding
- 训练更久
- 调大学习率到合理范围
- 检查数据和标签是否正确

### 4.2 过拟合

模型把训练数据背下来了，但泛化差。

表现：

```
train loss 很低
val loss 明显更高
```

常见解决：

- 增加数据
- 使用 dropout
- 加 weight decay
- 早停 Early Stopping
- 减小模型

### 4.3 梯度爆炸

参数更新过猛，训练不稳定。

表现：

```
loss 突然变成很大
甚至出现 NaN
```

常见解决：

- 降低学习率
- 使用梯度裁剪
- 检查输入是否异常

本项目里有：

```python
--grad_clip 1.0
```

### 4.4 梯度消失

梯度传到前面层时变得很小，模型很难学。

常见缓解：

- 使用残差连接
- 使用 LayerNorm
- 使用合适的激活函数
- 使用 Pre-Norm Transformer

本项目的 `Block` 使用的就是：

```python
x = x + self.attn(self.ln_1(x))
x = x + self.mlp(self.ln_2(x))
```

## 5. 常见超参数

| 参数 | 含义 | 太小会怎样 | 太大会怎样 |
|---|---|---|---|
| learning_rate | 每次更新多大 | 学得慢 | 震荡、不稳定 |
| batch_size | 每次看多少样本 | 梯度噪声大 | 占内存、可能泛化差 |
| n_layer | 网络层数 | 表达能力弱 | 训练更慢、更难 |
| n_embd | 向量维度 | 容量小 | 参数量大 |
| dropout | 随机丢弃比例 | 容易过拟合 | 学不动 |
| weight_decay | 权重衰减 | 容易过拟合 | 可能欠拟合 |
| block_size | 上下文长度 | 看不远 | 计算量增加 |

入门调参顺序建议：

```
先保证数据干净
再调 max_steps
再调 learning_rate
再调 batch_size
最后再扩大模型
```

## 6. 训练集、验证集、测试集

不要只看训练集效果。

```
训练集 train：用来更新参数
验证集 val：训练过程中观察泛化
测试集 test：最后一次评估，模拟真实效果
```

本项目 `TextBatcher` 默认把文本切成：

```python
train_split = 0.9
```

也就是：

```
90% 用于训练
10% 用于验证
```

## 7. 参数量怎么估算？

参数量 = 模型里所有可训练权重的数量。

简单估算：

```
Embedding 参数 ≈ vocab_size × n_embd
每层 Attention 参数 ≈ 4 × n_embd × n_embd
每层 MLP 参数 ≈ 8 × n_embd × n_embd
总参数 ≈ Embedding + n_layer × 每层参数
```

为什么 MLP 是 8 倍？

```
n_embd → 4*n_embd → n_embd
第一层：n_embd × 4*n_embd
第二层：4*n_embd × n_embd
合计：8 × n_embd²
```

这就是为什么 `n_embd` 一变大，参数量会涨得很快。
