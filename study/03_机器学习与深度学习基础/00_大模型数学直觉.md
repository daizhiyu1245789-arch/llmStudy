## 1. 大模型为什么离不开数学？

模型不认识文字。

模型真正处理的是数字：

```text
文字
  ↓ tokenizer
token ID
  ↓ embedding
向量
  ↓ Transformer
新的向量
  ↓ lm_head
每个 token 的分数
  ↓ softmax / sampling
下一个 token
```

所以大模型里的数学，本质是：

```text
用数字表示文本
用矩阵变换数字
用概率选择输出
用 loss 衡量错多少
用梯度更新参数
```

## 2. 标量、向量、矩阵、张量

### 2.1 标量 scalar

一个数字：

```text
3
0.8
-1.2
```

例子：

```text
temperature = 0.8
loss = 2.31
learning_rate = 0.0003
```

### 2.2 向量 vector

一串数字：

```text
[0.12, -0.33, 0.87, 0.01]
```

可以理解成“一个点的位置”或“一段信息的表示”。

在大模型里：

```text
一个 token → 一个向量
一句话 → 一个向量
一个文档 chunk → 一个向量
```

### 2.3 矩阵 matrix

二维数字表：

```text
[
  [1, 2, 3],
  [4, 5, 6]
]
```

可以理解成：

```text
很多向量排在一起
```

在大模型里：

```text
一批 token 的 embedding = 一个矩阵
线性层的权重 = 一个矩阵
Attention 分数 = 一个矩阵
```

### 2.4 张量 tensor

更高维的数字容器。

例如：

```text
[batch, seq_len, n_embd]
```

本项目里常见：

```text
x.shape = [batch_size, seq_len, n_embd]
```

意思是：

```text
一批样本
每个样本有一串 token
每个 token 是一个 n_embd 维向量
```

---

## 3. 为什么文本可以变成向量？

模型不能直接处理“天气”这个词。

它会把 token ID 映射成向量：

```text
"天气" → token ID → [0.12, -0.33, 0.87, ...]
```

这个向量不是人手写的，而是训练出来的。

训练后，语义相近的 token 或文本，向量往往更接近：

```text
苹果、水果、香蕉 → 接近
Node、Express、API → 接近
火锅、天气、数据库 → 不一定接近
```

这就是 Embedding 的基础。

---

## 4. 维度是什么？

一个 3 维向量：

```text
[0.2, 0.8, -0.1]
```

一个 768 维向量：

```text
[0.12, -0.33, ..., 0.09]  // 768 个数字
```

维度越高，能表达的信息通常越丰富。

但维度不是越大越好：

- 维度大，计算更贵
- 存储更大
- 检索更慢
- 需要更多数据训练

大模型里的 `n_embd` 就是 token 向量维度。

---

## 5. 矩阵乘法的直觉

神经网络大量使用矩阵乘法。

最常见：

```text
y = x @ W + b
```

你可以理解为：

```text
把输入向量 x，通过权重矩阵 W，变换成新的向量 y。
```

例子：

```text
原始 token 向量
  ↓ 线性层
Query 向量

原始 token 向量
  ↓ 线性层
Key 向量

原始 token 向量
  ↓ 线性层
Value 向量
```

在代码里就是：

```python
qkv = self.qkv(x)
```

一个线性层把 `x` 变成 Q、K、V。

---

## 6. 为什么线性层要加激活函数？

如果只有线性变换：

```text
y = x @ W
```

多层线性叠在一起，本质上还是一个线性变换。

所以需要激活函数引入非线性：

```text
Linear → ReLU/GELU → Linear
```

这样网络才能表达复杂关系。

Transformer 里的 MLP 就是：

```text
n_embd → 4*n_embd → GELU → n_embd
```

---

## 7. 概率分布是什么？

语言模型预测下一个 token 时，不是直接输出一个字，而是输出每个 token 的概率。

比如：

```text
输入：今天天气很

模型认为：
好：0.65
差：0.10
热：0.08
冷：0.05
其他：0.12
```

这些概率加起来等于 1。

这就是概率分布。

---

## 8. logits 是什么？

模型一开始输出的不是概率，而是一堆原始分数：

```text
好：3.2
差：1.1
热：0.8
冷：0.4
```

这些分数叫 logits。

logits 还不是概率：

- 可以是负数
- 不一定加起来等于 1
- 只是模型的“偏好分数”

需要经过 softmax 变成概率。

---

## 9. softmax 在干什么？

softmax 把一组分数变成概率分布。

```text
logits → softmax → probabilities
```

特点：

- 分数越大，概率越大
- 所有概率加起来等于 1
- 会放大高分和低分的差距

直觉：

```text
模型先给每个候选 token 打分
softmax 把打分变成“抽签概率”
```

---

## 10. temperature 是什么？

temperature 控制概率分布的平滑程度。

```text
temperature 低 → 更保守，更容易选最高概率 token
temperature 高 → 更随机，更有创意，也更容易乱
```

直觉：

```text
低温：模型更像考试答题
高温：模型更像头脑风暴
```

RAG 问答建议：

```text
temperature = 0.1 - 0.4
```

创意写作可以高一点：

```text
temperature = 0.8 - 1.2
```

---

## 11. loss 是什么？

loss 衡量模型预测错了多少。

语言模型常用交叉熵 loss。

直觉：

```text
正确答案是“好”

模型给“好”的概率越高 → loss 越小
模型给“好”的概率越低 → loss 越大
```

例子：

```text
P(好) = 0.9 → loss 小
P(好) = 0.1 → loss 大
```

训练目标：

```text
不断调整参数，让正确 token 的概率变高，让 loss 变小。
```

---

## 12. 梯度是什么？

梯度告诉模型：

```text
参数应该往哪个方向改，loss 会下降。
```

训练循环：

```text
前向传播：算预测
计算 loss：看错多少
反向传播：算梯度
优化器 step：更新参数
```

你可以把它理解成下山：

```text
loss 是山的高度
梯度告诉你哪里下坡最快
学习率决定每一步走多大
```

---

## 13. cosine similarity 是什么？

RAG 里常用余弦相似度比较两个向量是否相似。

直觉：

```text
看两个向量方向是不是接近。
```

方向接近：

```text
语义相似
```

方向差很远：

```text
语义不相似
```

所以可以用它做文档检索：

```text
用户问题向量
  ↓
和每个文档 chunk 向量算相似度
  ↓
取最相似的 topK
```

---

## 14. Attention 里的 QK^T 是什么？

Attention 公式：

```text
Attention(Q, K, V) = softmax(QK^T / √d) V
```

先不用怕。

直觉是：

```text
Q = 当前 token 想找什么
K = 每个 token 能提供什么索引
V = 每个 token 的实际内容
```

`QK^T` 在做：

```text
当前 token 和其他 token 的匹配分数
```

分数越高，说明越应该关注那个 token。

softmax 后变成注意力权重。

再乘 V，就是把相关 token 的信息加权拿回来。

## 15. 你需要掌握到什么程度？

入门阶段，你能解释这些就够了：

- 向量是一串数字，可以表示语义
- 矩阵乘法是在做向量变换
- logits 是模型输出的原始分数
- softmax 把分数变成概率
- loss 衡量预测错多少
- 梯度指导参数怎么更新
- cosine similarity 用来比较语义相似
- Attention 用相似度决定该看哪里

不用一开始就推公式。
