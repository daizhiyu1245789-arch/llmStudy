## 1. 机器学习的核心思想

### 一个公式总结机器学习

```
机器学习 = 自动从数据中找到"输入"和"输出"之间的规律
```

### 类比：JavaScript 里的函数拟合

假设你有一堆数据点：

```js
// 你知道输入和输出的对应关系
const data = [
  { x: 1, y: 2 },
  { x: 2, y: 4 },
  { x: 3, y: 6 },
  { x: 4, y: 8 },
];

// 你发现规律：y = 2 * x
// 然后你就能预测：
predict(5); // 10
predict(10); // 20
```

机器学习做的事情就是：**让计算机自己发现这个规律**。

```python
# 传统编程：人写规则
def predict(x):
    return 2 * x  # 人告诉计算机规则

# 机器学习：计算机从数据中学规则
model.fit(X_train, y_train)  # 从数据中自动学到规则
result = model.predict(X_test)  # 用学到的规则做预测
```

## 2. 机器学习的三个步骤

```
数据 → 训练（找规律）→ 模型 → 预测
```

### 步骤 1：准备数据

```python
# 训练数据 = 输入 + 期望输出
X = [[1], [2], [3], [4]]    # 输入 [1, 2, 3, 4]
y = [2, 4, 6, 8]            # 期望输出 [2, 4, 6, 8]
```

### 步骤 2：训练（让模型从数据中学规律）

```python
# 随机初始化一组参数
params = random_values()

# 喂数据给模型，看预测对不对
for epoch in range(1000):
    for x_i, y_i in zip(X, y):
        pred = model(x_i, params)     # 用当前参数做预测
        loss = (pred - y_i) ** 2     # 算错多远（损失函数）
        gradient = compute_gradient(loss, params)  # 算怎么改能减少错误
        params = params - lr * gradient  # 往错误减少的方向调参数

# 训练完后 params 应该 ≈ [2]（学到了 y = 2*x）
```

### 步骤 3：预测

```python
# 用训练好的参数做预测
predict(5)  # ≈ 10（2 * 5）
```

## 3. 深度学习 vs 传统机器学习

| | 传统机器学习 | 深度学习 |
|---|---|---|
| 特征谁提 | **人工设计**特征 | **自动学习**特征 |
| 模型结构 | 简单（线性、SVM、树） | 复杂（多层神经网络） |
| 参数量 | 少（几百～几万） | 多（百万～万亿） |
| 需要的数据 | 少（几千条够用） | 多（通常需要百万级） |
| 可解释性 | 较高 | 较低（"黑箱"） |
| 典型算法 | 线性回归、随机森林 | **神经网络、Transformer** |

### 本项目属于哪种？

**本项目 = 深度学习**。因为：
- 用的是 Transformer 架构（多层神经网络）
- 参数是从数据中自动学习的
- 不需要人工设计特征

## 4. 监督学习 vs 无监督学习

### 监督学习（Supervised Learning）

数据有"标签"，模型学习"输入→正确输出"的映射。

```python
# 有标签的训练数据
emails = ["免费赚钱", "你好聚会", "打折促销", "项目会议"]
labels =  [  "垃圾",   "正常",   "垃圾",     "正常" ]

# 模型学习：这段文字是垃圾还是正常？
model.fit(emails, labels)
model.predict(["限时优惠"])  # → "垃圾"
```

本项目 GPT 做的也是监督学习：
```python
# 输入："今天天气"   期望输出："很好"
# 训练时：让模型学会给定前文，预测下一个词
```

### 无监督学习（Unsupervised Learning）

数据没有标签，模型自己找规律。

```python
# 没有标签，模型自己发现：这几条数据可以分成两类
data = [[1, 2], [1.1, 2.1], [10, 10], [10.5, 10.5]]
clusters = model.fit_predict(data)
# 模型自动发现：前两个是一类，后两个是另一类
```

## 5. 什么是"训练"？

训练 = 反复做三件事：

```
1. 前向传播：输入数据，得到预测结果
2. 算损失：预测结果 vs 正确答案，算"错多远"
3. 反向传播 + 更新参数：调整参数让损失变小
```

```python
# 一个训练步骤（step）
for batch in data_loader:
    prediction = model(batch.input)          # 1. 前向传播
    loss = loss_fn(prediction, batch.label)  # 2. 算损失

    loss.backward()                          # 3. 反向传播（算梯度）
    optimizer.step()                         # 4. 更新参数
    optimizer.zero_grad()                    # 5. 清梯度，准备下一个 step
```

---

## 6. 什么是"损失函数"？

损失函数 = 衡量"预测值和真实值差多远"的函数。

```
损失大 = 错得离谱
损失小 = 预测接近真实值
```

常见的损失函数：

| 损失函数 | 公式 | 适用场景 |
|---------|------|---------|
| MSE（均方误差） | (预测-真实)² | 回归问题 |
| 交叉熵（Cross Entropy） | -真实概率 × log(预测概率) | 分类/语言模型 |
| MAE（绝对误差） | \|预测-真实\| | 回归问题 |

本项目用的是 **Cross Entropy（交叉熵）**：

```python
# 交叉熵在语言模型里的直观理解：
# "今天天气很好" → 下一个字最可能是"很"
# 模型预测"很"的概率 = 0.3 → cross_entropy = -log(0.3) = 1.2（损失大）
# 模型预测"很"的概率 = 0.9 → cross_entropy = -log(0.9) = 0.1（损失小）
```

## 7. 什么是"模型"？

模型 = 一个**参数化的函数**

```
输入 x + 参数 θ → 模型 → 输出 y
```

## 8. 欠拟合 vs 过拟合

### 欠拟合（Underfitting）

模型太简单，学不到数据中的规律。

```
训练 loss: 2.5   验证 loss: 2.8
→ 模型太笨，连训练数据都没学会
```

**解决**：增大模型（更多参数）、训练更久

### 过拟合（Overfitting）

模型死记硬背了训练数据，不泛化。

```
训练 loss: 0.01   验证 loss: 2.5
→ 模型把训练数据背下来了，但遇到新数据就不行
```

**解决**：加 Dropout、减少参数、增多训练数据

## 9. 机器学习项目的标准流程

```
1. 收集数据        → 准备训练语料
       ↓
2. 数据预处理     → Tokenize（文字→数字）
       ↓
3. 划分数据集     → 训练集 / 验证集
       ↓
4. 定义模型       → GPT 架构
       ↓
5. 选择损失函数   → Cross Entropy
       ↓
6. 选择优化器     → AdamW
       ↓
7. 训练循环       → 前向 → 损失 → 反向 → 更新
       ↓
8. 评估           → 验证集 loss
       ↓
9. 部署上线       → API / 模型文件
```

本项目的流程：
```
data/sample.txt → CharTokenizer → ids 数组
       ↓
DataConfig + TextBatcher → 批次采样
       ↓
GPT(GPTConfig) → 模型实例化
       ↓
AdamW 优化器 → 训练循环
       ↓
checkpoints/*.pt → 模型存档
       ↓
generate.py → 文本生成
```
