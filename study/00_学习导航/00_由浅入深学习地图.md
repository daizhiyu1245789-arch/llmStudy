# 由浅入深学习地图

> 这份地图只解决一个问题：资料很多时，先学什么、后学什么、哪些必学、哪些可以后补。

---

## 总体路线

```text
L0 学习导航
  ↓
L1 Node 工程基础
  ↓
L2 Python / PyTorch 运行基础
  ↓
L3 机器学习和深度学习原理
  ↓
L4 大模型核心：Tokenizer / Attention / GPT
  ↓
L5 Node 大模型应用实战：HF / Prompt / RAG / Tool Calling
  ↓
L6 扩展：论文 / Hugging Face 生态 / 部署优化
  ↓
练习 + 参考答案贯穿每一层
```

---

## L0：学习导航层

目标：先知道整套资料怎么用。

必读：

```text
00_学习导航/00_Node开发者学习总路线.md
00_学习导航/00_大模型术语表.md
00_学习导航/00_学习打卡和项目作业.md
```

读完你应该知道：

- 这套资料的最终目标是什么
- Node 开发者学大模型应该怎么走
- 遇到术语去哪查
- 每学完一章怎么练习

## L1：Node 工程基础层

目标：先打好做 AI 应用的工程底座。

目录：

```text
01_Node应用工程基础/
```

推荐顺序：

```text
1. README.md
2. 01_TypeScript工程基础.md
3. 02_HTTP接口和SSE流式响应.md
4. 03_环境变量和密钥安全.md
5. 04_异步编程和错误处理.md
6. 05_文件处理和文本预处理.md
7. 06_Node大模型应用项目结构.md
```

这一层解决：

- TypeScript 类型边界
- Zod 参数校验
- SSE 流式输出
- API Key 安全
- 异步错误处理
- 文件读取和文本切块
- Node 项目分层

为什么放在前面？

```text
因为后面的 Hugging Face、RAG、Tool Calling 都是 Node 工程能力的组合。
```

目录：

```text
02_Python和PyTorch基础/
```

推荐顺序：

```text

2. 02_Python独有概念.md
```

## L4：大模型核心层

目标：看懂 GPT 风格大模型的关键部件。

继续读：

```text
1. 04_Tokenizer分词器.md
2. 03_Attention机制.md
3. 05_大模型入门.md
```

这一层解决：

- 文本怎么变成 token
- token 怎么变成 embedding
- Attention 为什么能看上下文
- GPT 为什么只能看前文
- tinyllm 和真实大模型有什么对应关系

学习标准：

```text
能画出：
文本 → Tokenizer → Embedding → Transformer → logits → sampling → 文本
```

## 判断自己是否该进入下一层

每层都问自己三个问题：

```text
1. 我能用自己的话解释这一层的核心概念吗？
2. 我能完成对应练习吗？
3. 我能不用看答案，重新写一遍关键代码吗？
```

如果可以，就进入下一层。
