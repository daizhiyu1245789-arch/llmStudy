# 用自己的数据训练

> 这一篇教你用自己准备的语料训练一个模型。

---

## 1. 准备语料

### 格式要求

- 纯文本 `.txt` 文件
- 编码：**UTF-8**
- 内容：越干净越好（每行一段文字，或整个文件连在一起都可以）

### 语料来源建议

| 类型 | 说明 | 效果 |
|------|------|------|
| 小说/文章 | 叙事连贯 | 适合生成连贯故事 |
| 代码 | 编程问答/代码注释 | 适合代码补全 |
| 对话记录 | 聊天记录 | 适合对话生成 |
| 你的笔记 | 个人知识整理 | 适合个人知识助手 |
| 诗词/歌词 | 诗歌 | 适合生成诗词风格文字 |

### 语料清洗建议

```text
# 好的语料
今天天气真好，阳光温暖地照在窗台上，我泡了一杯咖啡，准备开始新的一天。

# 不好的语料（太多特殊符号和噪声）
[聊天记录] 2023-01-01 10:00:00 张三: 你好！@#$%^&*() 2023-01-01 10:00:01 李四: 嗨！

# 处理方法：提前把噪声字符去掉，只保留纯文字
```

### 语料大小

| 语料大小 | 建议训练步数 | 说明 |
|---------|-------------|------|
| 10KB（几千字） | 200-500 步 | 演示用，能看到学习效果 |
| 100KB（几万字） | 500-2000 步 | 最小可用规模 |
| 1MB（几十万字） | 2000-5000 步 | 能生成较流畅的文字 |
| 10MB+ | 5000+ 步 | 接近真实场景 |

**注意**：本项目是最小模型，不需要特别大的语料，10KB-100KB 足够看到效果。

## 2. 训练命令

### 基本训练

```bash
python -m tinyllm.train \
    --data_path 我的语料.txt \
    --max_steps 200 \
    --log_interval 20 \
    --eval_interval 50 \
    --save_interval 50
```

### 推荐参数配置

```bash
python -m tinyllm.train \
    --data_path 我的语料.txt \
    --tokenizer char \
    --n_layer 6 \
    --n_head 6 \
    --n_embd 256 \
    --block_size 128 \
    --batch_size 16 \
    --max_steps 500 \
    --log_interval 20 \
    --eval_interval 50 \
    --save_interval 100
```

参数对照：
```
n_layer=6    ← 6层 Transformer（默认4层效果更好）
n_head=6     ← 6个注意力头（默认4）
n_embd=256   ← 256维向量（默认128）
block_size=128 ← 128上下文长度
batch_size=16 ← 每批16个样本（显存不够时减小）
max_steps=500 ← 训练500步
```

## 3. 对比实验：不同参数的效果

### 实验 1：字符级 vs 字节级

```bash
# 字符级（默认，推荐中文）
python -m tinyllm.train --data_path 我的语料.txt --tokenizer char --max_steps 200

# 字节级（适合特殊文本、代码）
python -m tinyllm.train --data_path 我的语料.txt --tokenizer byte --max_steps 200
```

**结论**：中文用字符级，序列更短，效果更好。

### 实验 2：模型大小

```bash
# 小模型（快，适合快速验证）
python -m tinyllm.train --data_path 我的语料.txt --n_layer 2 --n_embd 64 --max_steps 200

# 大模型（慢，效果更好）
python -m tinyllm.train --data_path 我的语料.txt --n_layer 8 --n_embd 512 --max_steps 500
```

**结论**：大模型能学到更复杂的模式，但训练更慢。

### 实验 3：上下文长度

```bash
# 短上下文（只看最近的内容）
python -m tinyllm.train --data_path 我的语料.txt --block_size 32 --max_steps 200

# 长上下文（看更多内容）
python -m tinyllm.train --data_path 我的语料.txt --block_size 256 --max_steps 200
```

**结论**：长上下文生成更连贯，但需要更多显存。

## 4. 观察模型学习的过程

用不同 step 的 checkpoint 生成，观察模型"越学越聪明"：

```bash
# 第 20 步
python -m tinyllm.generate --ckpt_path checkpoints/step_20.pt --prompt "今天" --max_new_tokens 50

# 第 50 步
python -m tinyllm.generate --ckpt_path checkpoints/step_50.pt --prompt "今天" --max_new_tokens 50

# 第 100 步
python -m tinyllm.generate --ckpt_path checkpoints/step_100.pt --prompt "今天" --max_new_tokens 50

# 第 200 步
python -m tinyllm.generate --ckpt_path checkpoints/step_200.pt --prompt "今天" --max_new_tokens 50
```

你会看到：
```
step 20:  "今今今今今今今今今今今今今今今..."  （基本没学会）
step 50:  "今天今天今天今天今天今天..."  （开始学重复模式）
step 100: "今天天气好像有点不错..."  （开始学一些词）
step 200: "今天天气真是不错，阳光温暖地洒在街道上..."  （基本流畅）
```

## 6. 继续训练（断点恢复）

如果训练到一半中断了，可以继续：

```bash
# 从 latest.pt 继续训练
python -m tinyllm.train \
    --data_path 我的语料.txt \
    --resume \
    --max_steps 500
```

`--resume` 会自动加载 `checkpoints/latest.pt` 里的：
- 模型权重
- 优化器状态
- 分词器配置
- 当前 step 数

## 7. 常见问题

### Q：loss 降不下去怎么办？
A：
- 增大模型（n_layer、n_embd）
- 增加训练步数
- 检查语料质量（太短、太乱、太少）
- 尝试不同的学习率（--learning_rate 3e-4 或 1e-4）

### Q：生成重复内容怎么办？
A：
- 训练不够，增加 max_steps
- temperature 调低（0.7-0.8）
- top_k 调低（20-30）

### Q：loss 突然变成 nan 怎么办？
A：梯度爆炸。降低学习率：
```bash
python -m tinyllm.train --data_path 我的语料.txt --learning_rate 1e-4
```

### Q：显存不够（OOM）怎么办？
A：减小 batch_size 和 block_size：
```bash
python -m tinyllm.train --data_path 我的语料.txt --batch_size 4 --block_size 64
```
