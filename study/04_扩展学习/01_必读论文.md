## 论文 1：Attention is All You Need

**链接**：https://arxiv.org/abs/1706.03762

**中文翻译**：https://github.com/yzfly/Deep-Learning-Translation-Group/blob/main/Attention/Attention%20is%20All%20You%20Need.md

### 一句话总结

提出了 **Transformer** 架构，完全放弃了 RNN 和 CNN，只用 Attention 机制，在翻译任务上大幅超越以往所有方法。

### 核心内容

```
传统 seq2seq（RNN）：
输入 → RNN encode → 压缩向量 → RNN decode → 输出
                                    ↑
                            中间信息瓶颈问题

Transformer：
输入 → Encoder（6层 Attention）→ 输出
                        ↓
         Decoder（6层 Attention + Masked Attention）
                                    ↓
                               输出
```

### 在本项目中的对应

- **Multi-Head Attention** → `model.py` 的 `CausalSelfAttention`
- **Positional Encoding** → `model.py` 的 `wpe`（Position Embedding）
- **LayerNorm** → 每层的 `nn.LayerNorm`
- **Feed-Forward** → `model.py` 的 `MLP`

### 读这篇论文的重点

1. **Section 3** - 模型架构（图要能默写）
2. **Attention 公式** - `Attention(Q,K,V) = softmax(QK^T/√d_k)V`
3. **Positional Encoding** - 怎么把位置信息加进去

## 论文 2：GPT-2 Language Models are Unsupervised Multitask Learners

**链接**：https://d4mucfpksywv.cloudfront.net/big-scale/theverdict_3/papers/GPT2.pdf

### 一句话总结

证明了语言模型可以在**无监督预训练**后，通过**文本提示（prompt）**完成各种任务，无需针对每个任务单独训练。

### 核心创新

```
传统 NLP：每个任务一个模型
  - 翻译模型 → 翻译
  - 摘要模型 → 摘要
  - 问答模型 → 问答

GPT-2：一个大模型（语言模型）通吃所有任务
  - "翻译：法语英语..." → 翻译
  - "摘要：..." → 摘要
  - "问答：..." → 问答
```

### 在本项目中的对应

- **Causal LM（因果语言模型）** → 只用左侧上文预测下一个词
- **无监督预训练** → 直接在大量文本上训练语言模型
- **Text Generation** → `model.py` 的 `generate()` 方法
- **Decoding（temperature / top-k）** → `generate()` 里的采样逻辑

### 读这篇论文的重点

1. **Section 2** - 训练数据（WebText）
2. **Section 3** - 模型大小对比（Table 2 是经典）
3. **Section 4** - 各个任务上的效果（翻译、摘要、问答）

## 论文 3：GPT-3（选读）

**链接**：https://arxiv.org/abs/2005.14165

### 一句话总结

1750 亿参数的超大模型，展示了**Few-shot**（给几个例子就能做新任务）和**Zero-shot**（不给例子直接做）的能力，震撼业界。

### GPT 系列演进

| 模型 | 参数量 | 特点 |
|------|--------|------|
| GPT-1 | 1.17 亿 | 开创性 |
| GPT-2 | 15 亿 | 无监督多任务 |
| GPT-3 | 1750 亿 | Few-shot / Zero-shot |
| ChatGPT | ~1750 亿 | RLHF 对齐 |
| GPT-4 | 未公开 | 多模态 + 推理 |

### 本项目的模型规模

```
本项目：
  n_layer=4, n_head=4, n_embd=128
  参数量 ≈ 几十 MB

GPT-2：
  n_layer=48, n_head=25, n_embd=1600
  参数量 ≈ 1.5 GB

GPT-3：
  n_layer=96, n_head=96, n_embd=12288
  参数量 ≈ 350 GB
```

## 论文 4：BERT（选读）

**链接**：https://arxiv.org/abs/1810.04805

### 一句话总结

用 **Masked Language Model（MLM）** 替代传统语言模型，可以利用双向上下文，在理解任务上（如分类、问答）远超 GPT。

### BERT vs GPT 对比

| | BERT | GPT / GPT-2 |
|---|---|---|
| 预训练目标 | Masked LM（完形填空） | Causal LM（预测下一个） |
| 上下文 | 双向上下（能看到前后） | 因果（只看前文） |
| 适合任务 | 理解任务（分类、问答） | 生成任务（写作、对话） |
| 代表模型 | BERT-base: 110M | GPT-2: 1.5B |

```
BERT（双向）：
  "今天[MASK]气很好" → 能同时看到"今""天""很""好"
  → 预测"天"

GPT（单向）：
  "今天" → 预测"天"
  "今天天" → 预测"气"
  → 只能从左往右
```

## 读论文的技巧

### 1. 不用一次全看懂

论文里有大量公式推导和实验细节。第一次读：
- Abstract + Introduction（前两段）：搞清要解决什么问题
- Model 部分：搞清楚模型长什么样
- 实验部分：看效果就行，不用重复实验

### 2. 对照代码读

```
论文公式 → 本项目代码对照
```

本项目的 model.py 就是 GPT-2 的简化版实现。

### 3. 善用中文资源

很多经典论文已经有很好的中文解读：

| 论文 | 中文解读 |
|------|---------|
| Attention is All You Need | https://zhuanlan.zhihu.com/p/104393314 |
| GPT-2 | https://zhuanlan.zhihu.com/p/96757116 |
| BERT | https://zhuanlan.zhihu.com/p/364532996 |

### 4. 看完论文能回答这些问题

- [ ] 这个模型/算法的输入和输出是什么？
- [ ] 核心公式是什么？能默写吗？
- [ ] 和之前的方法相比，主要改进在哪？
- [ ] 在本项目（tinyllm）里，具体对应哪部分代码？
